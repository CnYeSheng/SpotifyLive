# 更新日誌 (Changelog)

## [v2.1.0] - 2024-12-19

### 🐛 修復問題 (Bug Fixes)

#### 1. 隨機播放按鈕狀態更新延遲
- **問題**: 隨機播放按鈕點擊後有反應，但關閉時變暗的更新時間過長
- **修復**: 
  - 優化按鈕狀態更新邏輯，減少延遲時間從1000ms到500ms
  - 添加即時視覺反饋，點擊時立即更新按鈕狀態
  - 改進錯誤處理，避免狀態不同步

#### 2. 重複播放按鈕狀態更新延遲  
- **問題**: 重複播放按鈕點擊後有反應，但關閉時變暗的更新時間過長
- **修復**:
  - 優化按鈕狀態更新邏輯，減少延遲時間從1000ms到500ms
  - 添加即時視覺反饋，點擊時立即更新按鈕狀態
  - 改進三種重複模式的視覺指示

#### 3. 播放清單載入問題
- **問題**: 播放清單載入不了，需要使用官方 API `https://api.spotify.com/v1/me/player/queue`
- **修復**:
  - 修復 `/api/player/queue` 端點的錯誤處理和速率限制
  - 改進隊列數據的解析和顯示，添加歌曲時長信息
  - 添加更好的錯誤提示和重試機制
  - 確保音樂點擊有正確反應，添加歌曲ID驗證
  - 新增 `/api/player/play` 端點支持播放指定歌曲
  - 新增 `/api/player/transfer` 端點支持設備切換

#### 4. 播放清單按鈕高亮顯示
- **問題**: 若這首歌有在播放清單，播放清單按鈕要亮起
- **修復**:
  - 添加當前歌曲在播放清單中的檢測邏輯
  - 實現播放清單按鈕的動態高亮狀態
  - 優化播放清單項目的當前歌曲標示

### 🚀 新增功能 (New Features)

#### 1. 改進的狀態同步
- 添加更快速的狀態更新機制
- 實現按鈕點擊的即時反饋
- 優化 API 調用頻率和時機
- 新增 `updatePlaylistButton()` 方法動態更新播放清單按鈕狀態

#### 2. 增強的播放清單功能
- 改進播放清單載入性能
- 添加當前播放歌曲的視覺標示
- 優化播放清單項目的點擊響應
- 新增歌曲ID驗證，防止無效歌曲播放
- 改進錯誤提示，提供更詳細的錯誤信息

#### 3. 新增 API 端點
- `/api/player/play` - 播放指定歌曲
- `/api/player/transfer` - 設備投放功能
- 兩個端點都包含完整的錯誤處理和速率限制

### 🔧 技術改進 (Technical Improvements)

#### 1. API 調用優化
- 減少不必要的 API 請求
- 改進錯誤處理和重試邏輯
- 優化狀態更新的時間間隔

#### 2. 用戶體驗改進
- 更快的按鈕響應時間
- 更清晰的視覺狀態指示
- 更穩定的播放清單功能
